# Contributing Guide

Thanks for your interest in contributing 🎉

We welcome contributions of all kinds:
- Bug fixes
- New features
- Documentation improvements
- Refactoring

If you're new to open source, this is a great place to start.

> ## New to Open Source?
> Start with issues labeled `good first issue`.  
> These are designed to be simple and well-guided.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Install dependencies

```bash
composer install
```
4. Run the project

```bash
php -S localhost:8000 -t public
```

## Finding Issues

We label issues to help you get started:

- `good first issue` → Beginner-friendly
- `help wanted` → Open for contributions
- `bug` → Fix something broken
- `enhancement` → Add or improve features

Start here:
👉 https://github.com/Faheem-maker/tiny-php-core/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22

## Working on an Issue

1. Comment on the issue to claim it
2. Wait for assignment (if required)
3. Create a branch:

```bash
git checkout -b fix/issue-name
```

4. Make your changes
5. Write/update tests if needed
6. Commit and push

## Submitting a Pull Request

- Ensure all tests pass
- Follow the PR template
- Keep PRs small and focused

Once submitted:
- A maintainer will review your PR
- Feedback may be provided
- After approval, it will be merged 🎉

## Code Guidelines

- Follow existing code style
- Write clear, readable code
- Add comments for complex logic
- Avoid unnecessary dependencies

## Need Help?

- Open a discussion
- Comment on issues
- Ask questions—we’re happy to help
